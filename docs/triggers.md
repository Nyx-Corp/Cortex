# Triggers — Multi-Modal Actions

Cortex v2 exposes every domain action through multiple triggers. The same `Command → Form → Handler → Response` pipeline runs regardless of the entry point.

## Architecture

```
         ┌─── HTML (Controller + FormType)
         │
Data ────┼─── CLI  (DomainActionCommand)
         │
         ├─── REST (ApiController)
         │
         └─── MCP  (ActionToolProvider)
                │
                ▼
         FormFactory->create($formType)
         $form->submit($data)
                │
                ▼
         CommandMapperExtension
         → new Command(...$data)
         → Handler->__invoke($command)
                │
                ▼
            Response
```

## HTML Trigger (existing)

Standard Symfony controller with `FormFactoryInterface`:

```php
$form = $this->formFactory->createNamed('name', ModelCreateType::class);
$form->handleRequest($request);
```

## CLI Trigger

Auto-registered via `ActionHandlerCompilerPass`. Naming convention: `{domain}:{model}:{action}`.

```bash
# List available commands
bin/console list studio

# Execute with options
bin/console studio:mannequin:create --firstname=Julie --gender=female

# JSON output
bin/console studio:mannequin:create --firstname=Julie --gender=female --format=json
```

## REST API Trigger

Routes auto-generated by `ApiRouteLoader`. Enable in your routes config:

```yaml
# config/routes/api.yaml
cortex_api:
    resource: .
    type: cortex_api
```

Requests:

```bash
# Create
curl -X POST /api/studio/mannequin/create \
  -H "Content-Type: application/json" \
  -d '{"firstname":"Julie","gender":"female"}'

# Update
curl -X PUT /api/studio/mannequin/{uuid} \
  -H "Content-Type: application/json" \
  -d '{"firstname":"Updated"}'

# Archive
curl -X DELETE /api/studio/mannequin/{uuid}/archive
```

Responses:
- `200` — success, serialized Response object
- `400` — validation errors with field-level details
- `422` — DomainException

## MCP Trigger

`ActionToolProvider` generates MCP tool definitions with JSON Schema input validation.

```php
$provider = $container->get(ActionToolProvider::class);

// List available tools
$tools = $provider->getTools();

// Execute a tool
$result = $provider->handleToolCall('studio_mannequin_create', [
    'firstname' => 'Julie',
    'gender' => 'female',
]);
```

Tool naming: `{domain}_{model}_{action}` (snake_case).
